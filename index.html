Opdracht 2.7 — file:// vs http:// (localhost:63342)

Er zijn twee manieren om een lokaal HTML-bestand in de browser te openen:

  • file:// — via dubbelklikken in de bestandsverkenner; de browser leest direct van je schijf. Geen webserver betrokken.
  • http://localhost:63342 — via een lokale webserver (bv. WebStorm of een eenvoudige dev-server). Hierdoor wordt de pagina via HTTP geserveerd en gelden dezelfde origin-/CORS-regels als op een echte server.

Belangrijk verschil: bij file:// kunnen sommige web-API's of relatieve padgedragingen anders werken (bv. fetch(), modules, sommige service workers). Werken via http://localhost is dichter bij echte productie-omstandigheden.

Opdracht 2.8 — Whitespace demonstratie

HTML negeert extra whitespace (meerdere spaties en regeleinden). Enkele manieren om extra ruimte te forceren:

  • &nbsp; — non-breaking space (forceert extra spatie)
  • <br /> — expliciete regeleinde

Voorbeeld:

<p>Eén  extra    spatie (meerdere spaties worden samengevat)</p>
<p>Woord1<br />Woord2 (hier is een expliciete nieuwe regel)</p>
<p>WoordA  WoordB (hier gebruiken we twee &nbsp;)</p>

Gebruik &nbsp; en <br /> spaarzaam — voor lay-out gebruik je CSS.

Opdracht 2.10 — Correct genest (haakjes)

Waarom zijn volgende uitdrukkingen niet correct genest?

  • (4x2 + 2( — er is een open haakje '(' zonder bijbehorende sluiting.
  • (4x2( +2 — zelfde probleem: er blijven ongepaarde open haakjes over.
  • (4x(2+2) — dit is correct genest: elke open haak heeft een bijbehorende sluiting in de juiste volgorde.
  • (4x(2+2() — onjuist: extra sluitingshaakje ontbreekt of de volgorde is verkeerd.

Algemene regel: bij het doorlopen van de tekst mag je nooit afsluiten met een ander haaktype dan het laatst geopende (LIFO-principe).

Opdracht 2.11 — Mix van haakjes () {} [] — correct genest?

Beantwoordingen:

  • ({}[]) — correct genest. Open '(', open '{' & close '}', open '[' & close ']', close ')'.
  • {{{()] — onjuist: er is een sluitende ']' voor de corresponderende '[' en de nesting wordt gebroken.
  • ({()}) — (in voorbeeld input stond '({()})' of '({()})'?) Als de vraag was ({()}) dan is het correct: ( { ( ) } ).
  • (){[}] — onjuist: het gedeelte {[}] is niet correct, de ']' sluit niet het laatst geopende '[' op de juiste manier.
  • {{([]) — onjuist: één of meerdere sluitende haakjes missen of de volgorde is verkeerd.

Kort: controleer altijd met een stack-model (laatste open haak wordt eerst gesloten).

Opdracht 2.12 — Foutieve nesting van <strong>

Gegeven fragment:

<span>hier komt een</span><a href="">link</a><span> te staan</span>

Auteur wil 'een link' benadrukken en veranderde naar:

<span>hier komt <strong>een</span><a href="">link</a></strong><span> te staan</span>

Waarom fout? Omdat tags niet correct genest zijn: de <strong> tag wordt geopend binnen de eerste <span> maar gesloten ná de <a> tag — dat breekt de LIFO-regel.

Correcte versie — twee mogelijkheden:

<!-- Optie A: Voeg <strong> rond de volledige tekst <span>een link</span> -->
<span>hier komt <strong>een <a href="">link</a></strong> te staan</span>

<!-- Optie B: Plaats <strong> uitsluitend om de link zelf -->
<span>hier komt een <strong><a href="">link</a></strong> te staan</span>

Belangrijk: zorg dat tags correct genest zijn — sluit eerst de laatst geopende tag.

Opdracht 2.14 — Wat is verkeerd in het HTML-fragment?

<ul>
  <li>een</li>
  <li>twee</li>
  <ol>
    <li>jan</li>
    <li>piet</li>
    <li>joris</li>
  </ol>
  <li>vier</li>
</ul>

Probleem: een <ol> is geopend binnen een <ul> zonder juiste nesting of afsluiting van de <ul> list. Als het bedoeling is een genest nummerlijk lijstje te hebben, moet het <ol> genest worden binnen een <li> of de lists moeten correct gesloten worden.

Correct voor geneste lijst:

<ul>
  <li>een</li>
  <li>twee
    <ol>
      <li>jan</li>
      <li>piet</li>
      <li>joris</li>
    </ol>
  </li>
  <li>vier</li>
</ul>

Opdracht 2.15–2.17 — Website vs Webapplicatie (samenvatting)

Website: verzameling documenten, voornamelijk informatief. Kan statisch (HTML-bestanden) of dynamisch (gegenereerd per request) zijn.

Webapplicatie: biedt functionaliteit en complexe interactie (bv. winkelwagen, gebruikersprofielen). Data wordt vaak zowel in de browser als op de server opgeslagen, afhankelijk van het doel en veiligheidseisen.

Belangrijk: gevoelige gegevens altijd server-side bewaren; client-side opslag (localStorage, cookies, indexedDB) is makkelijk maar niet veilig tegen manipulatie of lokale toegang.

Opdracht 2.18 — Microsoft & local storage / cookies

Wat je ziet in Chrome Developer Tools (Application tab):

  • Local Storage: key-value opslag die persistent blijft voor hetzelfde origin en niet automatisch met elke request naar de server gestuurd wordt.
  • Cookies: worden vaak gebruikt voor sessies, voorkeuren en small state; cookies worden standaard meegestuurd bij HTTP requests (afhankelijk van domain, path en flags).

Als je de regiotaal van een site wijzigt, wordt dat vaak opgeslagen in een cookie of in localStorage — kijk in DevTools onder de betreffende domain-entry om exact te zien waar (soms wordt ook server-side voorkeur ingesteld en aangevuld met een cookie).

Opdracht 2.19 — LG & Amazon (localStorage / cookies / winkelwagen)

LG-vergelijkfunctie: veel sites bewaren tijdelijke selectie (bv. welke items je hebt aangevinkt) in localStorage of in cookies zodat de selectie over meerdere pagina's behouden blijft. Open DevTools > Application > Local Storage om de key-value paren te inspecteren.

Amazon en winkelwagen zonder inloggen:

  • Als je niet ingelogd bent, gebruikt Amazon meestal een combinatie van een server-side winkelwagen gekoppeld aan een unieke identifier en een cookie (bv. session cookie) om die identifier op jouw browser te bewaren.
  • Dat betekent: bij toevoegen van een product verstuurt de server een ID naar jouw browser (cookie). Later kan Amazon die cookie lezen en de corresponderende winkelwagen ophalen op de server.

Kort: vaak is de winkelwagen niet volledig in localStorage/cookies opgeslagen — alleen een verwijzing (ID) die de server naar de juiste server-side data laat verwijzen.
